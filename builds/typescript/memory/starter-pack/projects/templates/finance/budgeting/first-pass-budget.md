# First-Pass Budget Workflow

Use this when the owner asks to create a budget, revise the saved budget, set category limits, understand spending from uploaded statements, or get a first useful version of their monthly spending plan.

## Goal

Create a usable saved budget the owner recognizes as theirs: stable categories, monthly limits, important fixed bills, and any current spending goals such as debt payoff or savings.

Do not wait for perfect information. A provisional but useful budget is better than no budget.

## Inputs

Read:

- `../budget.md` for the current saved plan, if any.
- `../rules.md` for owner-approved categorization and transaction-type rules.
- `source-evidence.md` if uploaded statements are available or the owner asks to use statements.
- Relevant files under `../statements/` when statement evidence is needed.
- `../spec.md` and `../plan.md` when broader goals matter.

## Behavior

- Use owner estimates, uploaded statements, or both.
- Label assumptions and uncertain items plainly.
- Put ambiguous merchants or missing context in needs review instead of inventing precision.
- Keep categories stable where possible so future reports can compare against the same saved plan.
- Add or change categories only when the owner understands the change and it fits how they want to think about spending.
- Emotional or relationship context can inform the budget, but do not let it replace the budget artifact when the owner asked for numbers.
- If partner context matters, add a brief note or follow-up after the budget artifact.

## Output

When creating or revising the saved budget:

- Update `../budget.md` with the saved monthly plan.
- Show the useful budget artifact in chat: category limits, observed actuals when available, fixed bills, confidence, assumptions, and needs-review items.
- Do not only say that the file was saved.

Treat a provisional budget as usable work. It is okay to say the numbers are not final, but do not frame the budget as merely a conversation starter after showing the requested draft.
