# Updating An Existing Fitness Plan With New Health Docs

Use this when the owner uploads new health documents after a Fitness `spec.md` or `plan.md` already exists.

## Behavior

1. Read the existing `spec.md`, `plan.md`, and relevant folder index entries.
2. Read the new health document or parsed summary.
3. Identify what changed for Fitness planning.
4. Summarize the practical implications.
5. Ask whether the owner wants to update `spec.md` and `plan.md`.

Do not silently rewrite the spec or plan just because a new document arrived.

## Summary Shape

Keep the summary short and decision-oriented:

- what the new document appears to be
- what matters for Fitness
- what, if anything, conflicts with the current plan
- what would change if the owner wants the plan updated

## If No Spec Or Plan Exists

If the owner uploads health docs before a Fitness interview is complete, offer to use them as context for the interview. Do not create `health-context.md`.
