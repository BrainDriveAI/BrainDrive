## What makes this spec good (the "hill to climb")

A good spec for any project passes these tests:

- **User stories use the owner's exact words** — quote, don't paraphrase. The owner reads them and thinks "yes, that's exactly what I want and why."
- **Where You Are reflects the actual current state** — what exists, what resources are real, who's involved. Adapt the focus to project type (creative project: what's been written; renovation: scope and budget; life change: emotional + practical landscape).
- **Obstacles are specific** — "I open the document and immediately want to do anything else" beats "procrastination." "$30K budget but quotes coming back at $45K" beats "money is tight."
- **Gaps separated by impact** — "before the plan is complete" (could change direction) vs "worth exploring later."

## Interview methodology — How to fill this spec

When the spec is `Pre-interview` or `Partial`, run an interview to fill it. This methodology applies broadly across all domains.

### Universal interview rules

- **One question at a time.** Listen to the answer. Follow up on what matters. Do NOT bundle 3+ questions in one message — that's a form, not a conversation.
- **Landscape first, specifics second.** Owner usually comes with a specific situation. Build the picture first, circle back later.
- **Never accept vague answers.** Probe for specifics.
- **Confirm as you go.** Play back high-stakes sections.
- **Confirm user stories before writing the plan.**
- **~5 minutes is the target.** Adapt to the project — life crises and creative projects often need more time.
- **The user stories in this spec are the most important output.**

### Adapting to the project type

You don't have domain-specific seeds (this is the catch-all). Adapt the structure to what the owner brings.

**Where You Are varies wildly by project type:**
- Home/practical project (renovation, trip planning) → scope, budget, timeline, who's involved
- Creative project (writing, painting, music) → what exists already, what's stuck
- Learning project (a new skill) → current level, time commitment, motivation
- Life transition (move, divorce, job change, processing grief) → emotional + practical landscape both matter

**Common blind spots by project type:**
- Creative projects → overthinking, perfectionism, waiting for inspiration instead of starting
- Practical projects (home, travel) → underestimated complexity, unclear budget, scope creep
- Learning projects → no definition of "good enough," trying to master before practicing
- Life transitions → conflating urgency with importance, trying to solve everything at once

**When multiple interests surface** (e.g., drawing AND baking, writing AND photography), probe connections between them before choosing which to develop. The first-mentioned isn't always the most important.

### Handling sensitive / personal topics

If the owner brings something deeply personal (grief, addiction, life crisis):

- Be honest: "This is deeply personal and doesn't fit a neat template. Let's figure out what progress looks like for you."
- The methodology still works — "What does success look like?" applies to grief as much as a renovation.
- May need more time — be upfront: "This deserves more than a quick start. Want to spend more time on it?"
- Always ask about support systems — who else knows, who could help (emotionally or practically).
- For acute crisis (self-harm, abuse): follow `base/AGENT.md` "Sensitive Topics — Escalation, Not Filing" — STOP the interview, acknowledge, suggest professional support, capture only after.

### When the spec is "enough" — completion checklist

The spec is ready to commit (status → `Complete`) when ALL these are true:

- [ ] At least one user story captured in owner's exact words under "What You Want"
- [ ] "Where You Are" reflects the actual current state (adapted to project type)
- [ ] "What's In The Way" surfaces at least 1 specific obstacle
- [ ] Owner has confirmed the user story(s) by playback
- [ ] Immediate first step is nameable

If any false, keep interviewing. If all true, write spec, write plan, flip AGENT.md status.
